## <u>Background</u>: "개발 협업의 숨겨진 장벽"

### 🎯 1. 프로젝트 필요성

* **높은 학습 진입 장벽:** 프로그래밍 입문자나 비전공자에게 Git CLI(명령줄 인터페이스)는 수십 개의 명령어를 암기해야 하는 매우 복잡하고 높은 학습 장벽으로 작용합니다.
* **협업의 어려움:** 실제 현업의 필수 역량인 Pull Request, 코드 리뷰, 브랜치 전략 등의 협업 워크플로우를 경험할 기회가 부족하며, 특히 '병합 충돌(Merge Conflict)'은 초보자의 학습 의지를 저하시키는 가장 큰 요인입니다.
* **교육의 한계:** 기존 교육은 Git의 '개념'이나 '명령어' 암기에 치중되어 있어, 실제 프로젝트에서 '어떻게 협업하는지'에 대한 실질적인 경험을 제공하지 못합니다.

### 🌪️ 2. 기존 해결책의 문제점

* **CLI (명령줄 인터페이스):** 입문자가 사용하기에는 극도로 불친절하며, 사소한 실수(예: `git push -f`)가 프로젝트 전체를 위험에 빠뜨릴 수 있습니다.
* **기존 GUI 도구 (예: SourceTree):** CLI를 시각화해주지만, 여전히 Git의 복잡한 내부 동작(예: 병합 충돌)을 사용자가 직접 해결해야 하며, 협업 기능(코드 리뷰 등)이 통합되어 있지 않습니다.
* **GitHub / GitLab 플랫폼:** 강력한 협업 기능을 제공하지만, 초보자가 플랫폼의 모든 기능을 이해하고 실습하기에는 너무 방대하며, 근본적인 충돌 해결의 어려움은 여전히 남아있습니다.

## <u>System Design</u>: "AI로 똑똑하게, GUI로 쉽게"

### 📋 1. 핵심 요구사항 (System Requirements)

* **직관적인 GUI 제공:** 사용자는 CLI 명령어 없이 모든 Git 작업(커밋, 브랜치 생성, 푸시, 풀)을 웹 UI 상에서 수행할 수 있어야 합니다.
* **AI 기반 지능형 충돌 해결:** 병합 충돌 발생 시, **Claude AI**가 코드 의도를 분석하여 최적의 병합 안을 '제시'하고 '설명'할 수 있어야 합니다.
* **실전 협업 워크플로우 지원:** Pull Request 생성, 동료의 코드 리뷰(댓글) 및 승인, 병합에 이르는 실제 협업 사이클을 완벽하게 지원해야 합니다.
* **시각적 히스토리 제공:** 커밋과 브랜치의 흐름을 시각적인 그래프로 제공하여 버전 관리 흐름을 쉽게 파악할 수 있어야 합니다.

### 🛠️ 2. 기술 스택 (Tech Stack)

| 구분 | 기술 스택 | 설명 |
| :--- | :--- | :--- |
| **Backend** | **NestJS, TypeORM, PostgreSQL** | 안정적인 API 서버 및 메타데이터 관리 |
| | **`simple-git` (Node.js)** | 서버 환경에서 Git 명령어를 제어 |
| | **Claude AI API** | 지능형 병합 충돌 해결 엔진 |

| **Frontend** | **React, JavaScript (ES6+)** | 컴포넌트 기반의 동적 UI/UX 구현 |
| | **RESTful API** | 백엔드와의 비동기 데이터 통신 |

## <u>Case Study</u>: "비전공자 '나초보'의 협업 도전기"

### 📖 시나리오 설명 (Description)

1.  **[시작]** '나초보' 학생은 'Teamate' 플랫폼에서 GUI 버튼 클릭만으로 새 저장소를 생성하고 팀원을 초대합니다.
2.  **[개발]** '로그인 기능' 브랜치를 생성하고, UI 상에서 코드를 작성한 뒤 '커밋' 버튼을 눌러 작업을 저장합니다.
3.  **[리뷰]** 'Pull Request'를 생성하여 팀원 '이개발'에게 코드 리뷰를 요청합니다. '이개발' 학생은 코드 변경 사항을 확인하고 "이 부분은 수정이 필요해요"라고 댓글을 남깁니다.
4.  **[충돌]** 그 사이 '이개발' 학생이 수정한 코드가 메인 브랜치에 병합되었고, '나초보' 학생이 PR을 병합하려 하자 **'병합 충돌'**이 발생합니다.
5.  **[해결]** '나초보' 학생은 당황하지 않고 **'AI로 충돌 해결하기'** 버튼을 누릅니다. **Claude AI**가 두 코드의 의도를 분석하여 "로그인 로직은 유지하되, 메인의 보안 패치를 적용한" 최적의 병합 코드를 제시합니다. '나초보' 학생은 AI의 설명을 읽고 '적용' 버튼을 눌러 충돌을 손쉽게 해결합니다.

## <u>Conclusion</u>: "누구나 할 수 있는 개발 협업"

### 💡 1. 프로젝트 의의

본 프로젝트는 Git의 복잡성을 AI와 직관적인 GUI로 해결하여, 프로그래밍 입문자와 비전공자의 **개발 협업 진입 장벽을 획기적으로 낮추는 '교육용 협업 플랫폼'**의 가능성을 제시합니다. 학생들은 명령어 암기 대신, '왜' 협업 워크플로우가 필요한지 문제중심학습(PBL) 방식으로 체득할 수 있습니다.

### ➡️ 2. 향후 계획 (Future Plans)

* **AI 코드 리뷰어:** AI가 병합 충돌뿐만 아니라, 코드 스타일이나 간단한 논리적 오류를 사전에 검토하고 제안하는 'AI 코드 리뷰' 기능을 추가 개발할 계획입니다.
* **실제 교육 적용:** 한밭대학교 프로그래밍 입문 수업에 파일럿 테스트로 적용하여, 실제 학생들의 사용성 피드백을 받아 서비스를 고도화할 것입니다.
